# NTSI 專案強制開發規範

## 開始工作前

任何涉及人物、NPC、玩家外觀、服裝、劇情、對話演出、UI 人像或生成美術資產的任務，在分析或動工前必須完整閱讀：

1. `docs/art/CHARACTER_PORTRAIT_STANDARD.md`
2. `docs/art/角色美術與境界演出母板.md`
3. `art/README.md`
4. `game/src/config/characters.ts`
5. `docs/art/UI_STYLE_GUIDE.md`

本步驟不可略過；若規格與使用者最新指示衝突，以使用者最新指示為準，並同步更新規格。

## 人物與對話

- 場景需要人物站位或動作時使用完整立繪；不需要完整身形時使用符合母板的胸像頭像。
- 禁止以圓形、三角形、矩形或其他程式幾何人偶代替已有角色。
- 角色名稱不得使用【】包圍；系統提示與角色對話必須使用不同 HUD 樣式。
- 小遊戲選擇卡只顯示玩法名稱與必要狀態，不得顯示「點擊開始」等多餘操作文字；互動性以游標、觸控回饋與動效表達。
- 玩家男女角、既有 NPC 與所有新角色都必須納入角色資產清單。
- 角色更換服裝、身份或境界時，必須建立獨立服裝版本及該版本所需表情頭像，不得覆蓋舊版本。

## 生成與新增美術

- 新人物必須先建立完整角色母板、中性立繪、同框比例驗收與 neutral 頭像，再製作其他表情。
- 新表情只可調整眉、眼、嘴及合理的細微神態；臉型、五官位置、髮型、服裝、鏡頭與裁切必須一致。
- 圖片內不得生成遊戲文字；繁體中文由遊戲 UI 即時渲染。
- 原始圖放入 `art/` 對應分類；遊戲執行副本放入 `game/public/assets/`。
- 新增、改名或搬移資產時，必須同步更新角色清單、交接文件與資產搬移對照。
- 活動劇情場景卡必須依玩家性別與當前境界使用已驗收服裝版本；不同境界可升級卡片演出，但不得自行改變小遊戲操作規則。
- 所有遊戲場景、場景卡與 UI 母板一律以 16:9 橫式設計；舊 9:16 圖只能作為內容參考，不得裁切、拉伸或宣稱為正式橫式素材。
- 首圖與核心常駐場景必須規劃卯時、午時、酉時三個同鏡頭版本；小遊戲在玩法與基準美術定案前不得提前生成午時、酉時版本。
- UI 與切換流程母板只能作為實機配置、UI、UX、HUD 及互動參考，禁止整張放入遊戲或裁成一張假介面。
- 禁止使用圖片生成取代 HUD 圖層、按鈕、進度條、功能圖示、文字、互動狀態、頁面切換或程式功能。
- UI 必須拆成可獨立維護的場景、人物、面板、框線、圖示、按鈕狀態、進度條與文字圖層；互動與數值由程式實作。
- 小遊戲基礎時限為 30 秒並必須顯示倒數；時間到未達成功條件一律標記失敗。
- 母板中的「宗門事務」與「散步遊歷」不是實機功能，禁止列入開發或資產製作。

## 完成條件

- 未通過母板、透明背景、同框比例、表情一致性及手機顯示驗收的素材，只能標記為候選，不得宣稱正式完成。
- 修改程式後必須執行 `npm run validate:balance` 與 `npm run build`。
