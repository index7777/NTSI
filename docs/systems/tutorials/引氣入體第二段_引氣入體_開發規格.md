一、第二段的核心體驗

玩家不是把一顆 UI 球拖進目標圈。

玩家是在：

順著靈氣自身的流勢，將聚集完成的天地靈氣引向角色丹田。

靈氣必須具有：

慣性
拖尾
聚散
輕微漂移
靠近丹田時的吸附感

操作應簡單、寬容、必定成功。

五、操作方式

桌面：

滑鼠按住靈氣團
拖曳引導
放開後可再次接續

觸控：

手指按住靈氣團
拖曳引導

使用 Pointer Events：

pointerdown
pointermove
pointerup
pointercancel

不要另外維護 mouse 和 touch 兩套邏輯。

六、起始靈氣團

靈氣團位置：

const QI_START = {
  x: 0.31,
  y: 0.53
};

丹田位置：

const DANTIAN_TARGET = {
  x: 0.515,
  y: 0.565
};

這些為相對畫面比例，必須依實際角色圖微調。

靈氣團視覺：

淡青白為主。
混入少量淡金。
中央不是實心白球。
由約 12～20 個粒子與柔和霧團組成。
外圍緩慢旋轉。
每 2～3 秒有極小幅度呼吸收縮。
尺寸約為角色頭部的 0.8～1.1 倍。
不要像火球或戰鬥技能。
七、引導路徑

玩家不是自由拖到任何地方。

畫面存在一條低調的彎曲引導路徑：

靈氣團
   ↘
     旋轉彎曲
          ↘
            丹田

建議使用二次或三次 Bézier 曲線。

路徑：

const GUIDE_PATH = {
  start: { x: 0.31, y: 0.53 },
  control1: { x: 0.39, y: 0.43 },
  control2: { x: 0.47, y: 0.50 },
  end: { x: 0.515, y: 0.565 }
};

畫面上只顯示：

低透明度淡金或淡青線。
路徑局部亮起。
不要顯示粗箭頭。
不要像教學連線題。
八、拖曳不是直接跟手

禁止：

qiX = pointerX;
qiY = pointerY;

靈氣團必須有延遲與慣性。

建議：

const followStrength = 7.5;
const damping = 0.84;

velocityX += (pointerX - qiX) * followStrength * deltaTime;
velocityY += (pointerY - qiY) * followStrength * deltaTime;

velocityX *= Math.pow(damping, deltaTime * 60);
velocityY *= Math.pow(damping, deltaTime * 60);

qiX += velocityX * deltaTime;
qiY += velocityY * deltaTime;

要求：

游標移動時，靈氣團稍慢跟隨。
停下時，靈氣團會微微越過再回穩。
延遲不能太重，避免操作挫折。
手感應像引導煙霧，不像搬移圖示。
九、路徑吸附

玩家可以稍微偏離路徑，但系統會將靈氣團柔和拉回。

先求出目前離引導路徑最近的點：

closestPointOnGuidePath

再加入吸附：

const PATH_MAGNET_STRENGTH = 2.8;

velocityX +=
  (closestPoint.x - qiX) *
  PATH_MAGNET_STRENGTH *
  deltaTime;

velocityY +=
  (closestPoint.y - qiY) *
  PATH_MAGNET_STRENGTH *
  deltaTime;

偏離路徑時：

靈氣團稍微散開。
光芒降低。
引導線顯示稍微清楚。
不扣分。
不重置。
不出現失敗訊息。

回到路徑後自動恢復聚合。

十、放開後的行為

玩家提早放開時：

靈氣團停在附近。
保留目前路徑進度。
受到輕微漂移。
緩慢向最近的有效路徑點靠攏。
不退回起點。
不判定失敗。
玩家再次按住即可繼續。

狀態文字改為：

再次按住，繼續引導靈氣

不要在放開時立即消散所有粒子。

十一、拖尾與聚散

靈氣移動時，產生 8～14 個短生命週期粒子。

interface QiTrailParticle {
  x: number;
  y: number;
  velocityX: number;
  velocityY: number;
  opacity: number;
  lifetime: number;
  size: number;
}

拖尾要求：

比主靈氣團更透明。
壽命約 0.4～0.9 秒。
緩慢縮小。
輕微向上漂移。
不形成粗大光帶。
不遮住角色臉部。

移動越快，拖尾稍長；停下時逐漸收束。

十二、丹田引導標記

丹田位置顯示低調標記：

細圓環
內部一個小光點
非常淡的呼吸明暗

不得使用：

八卦陣
法陣符文
經脈透視
大型金色魔法圈
巨大「丹田」文字

初始透明度：

0.22

靈氣越靠近丹田，透明度與尺寸略微增加：

opacity = lerp(0.22, 0.62, proximity);
scale = lerp(1, 1.08, proximity);
十三、接近丹田時的吸附

當距離丹田小於：

const ABSORB_RADIUS = 0.065;

進入 absorbing。

此時：

玩家可繼續拖曳，但系統主導吸收。
靈氣團繞丹田約半圈。
尺寸逐漸縮小。
粒子沿曲線匯入丹田。
角色腹部產生低強度明暗呼吸。
衣帶與髮絲的動態略微增強。
背景色溫稍微提高。
不要瞬間「啪」地消失。

吸收時間：

const ABSORB_DURATION = 1.6;

吸收結束後：

phase = "completed";

銜接第三段「氣機相合」。

十四、吸收動畫時間線
0.0～0.4 秒
靈氣團減速並靠近丹田

0.4～1.0 秒
繞丹田半圈，拖尾向內收束

1.0～1.4 秒
主光團縮小，大部分粒子被吸入

1.4～1.6 秒
丹田柔和亮起，剩餘微光消失

不得加入全畫面閃白。

十五、畫面色彩變化

第二段開始時：

仍保留灰綠、水墨、低飽和

隨著靈氣靠近丹田：

角色附近光線微暖。
淡青靈氣稍微混入米金。
石台與服裝亮度提高約 5%～8%。
背景霧層仍保持冷灰。

只讓角色附近變化，避免整張圖直接變黃。

十六、角色動態

若角色有獨立透明圖層：

呼吸
3.2 秒一循環
極小幅度上下移動與縮放
靠近丹田

隨 proximity 增加：

衣帶擺幅提高約 10%～18%。
頭髮末端輕微向外飄。
袖口有非常小的抬起感。
手掌附近出現少量細微粒子。

如果只有單張合成角色圖：

不要縮放整個背景。
只做粒子、局部光暈與前景竹葉。
後續再換分層角色素材。

玩家停留超過 4 秒未操作：

靈氣團朝正確方向輕微漂移。
引導路徑稍微增亮。
丹田標記輕微脈動。
顯示一次簡短提示。

玩家反覆偏離路徑：

增強路徑吸附。
擴大有效區域。
不顯示負面文字。

必須保證所有玩家都能完成。

十九、狀態機
switch (phase) {
  case "intro":
    // 播放第二段進場與靈氣聚合

    // 等待玩家按住靈氣團
    break;

  case "dragging":
    // 更新輸入、慣性、路徑吸附、拖尾
    break;

    // 保留進度，靈氣緩慢漂移
    break;

  case "absorbing":
    // 鎖定主要控制，播放丹田吸收動畫
    break;

  case "completed":
    // 銜接第三段
    break;

  case "paused":
    // 停止所有動畫與輸入
    break;
}

二十二、驗收標準

完成後必須符合：

與第一段使用相同背景、角色與鏡頭。
第一階段標記為完成，第二階段高亮。
靈氣團不是實心發光球。
拖曳時具有輕微慣性，不完全黏著游標。
靈氣團會被引導路徑柔和吸附。
偏離不會失敗或重置。
放開後保留進度。
再次按住可從原處繼續。
靠近丹田會自動進入吸收動畫。
吸收過程不少於約 1.5 秒。
不使用 Perfect、Good、Miss。
不顯示時間、連擊、修為獎勵。
暫停會停止粒子、動畫和狀態更新。
手機與桌面均可操作。
完成後只觸發一次第三段。
Console 無錯誤。
請不要把本段做成普通 drag-and-drop。
玩家拖動的是具有慣性、聚散與流動感的天地靈氣。
靈氣應受到玩家引導，但不能完全黏住游標。
偏離路徑時只提供柔和修正，不設失敗。
靠近丹田後，由系統播放自然吸收動畫，再銜接第三段。

本段的重點不是操作難度，而是讓玩家感覺自己親手完成第一次引氣入體。
