《九劫問仙》引氣入體第一段：感應靈氣

請先閱讀目前專案，沿用現有遊戲畫面的路由、UI 尺寸規則、暫停系統和輸入管理。

本次任務是實作一次性劇情教學：

引氣入體—第一段：感應靈氣

這不是 25 秒計分小遊戲，不顯示修為獎勵、連擊、剩餘修行次數或評級。

請使用以下無 HUD 靜態圖片作為畫面底圖：

/assets/cultivation/sensing-spirit/sensing_spirit_bg.png

所有 HUD、靈氣與動態效果必須由程式分層產生，不能烘焙在圖片中。

一、玩法狀態
type SensingSpiritPhase =
  | "intro"
  | "waiting"
  | "holding"
  | "released"
  | "completed"
  | "paused";

核心資料：

interface SensingSpiritState {
  progress: number;       // 0～1
  isHolding: boolean;
  holdDuration: number;
  visualIntensity: number; // 0～1，畫面效果強度
}
二、輸入方式

桌面：

滑鼠左鍵按住
Space 按住

行動裝置：

觸控畫面並按住

按下時：

isHolding = true;

放開時：

isHolding = false;

不要將 click 當成一次性輸入，必須使用：

pointerdown
pointerup
pointercancel
keydown
keyup

必須處理滑鼠移出視窗、切換頁籤與失焦，避免卡在按住狀態。

三、進度邏輯

建議完整按住時間：

const HOLD_TO_COMPLETE_SECONDS = 5.5;

按住時：

progress += deltaTime / HOLD_TO_COMPLETE_SECONDS;

提早放開時，先停頓 0.25 秒，之後才非常緩慢回退：

const RELEASE_GRACE_SECONDS = 0.25;
const DECAY_PER_SECOND = 0.035;

邏輯：

if (isHolding) {
  progress += deltaTime / HOLD_TO_COMPLETE_SECONDS;
  timeSinceRelease = 0;
} else {
  timeSinceRelease += deltaTime;

  if (timeSinceRelease > RELEASE_GRACE_SECONDS) {
    progress -= DECAY_PER_SECOND * deltaTime;
  }
}

progress = clamp(progress, 0, 1);

不得：

一放開就歸零
判定失敗
跳出 Miss
重新開始整段

當 progress >= 1：

phase = "completed";

並鎖定輸入，銜接第二段「引氣入體」。

四、圓形蓄力進度

角色下方或丹田前方顯示圓形蓄力環。

使用 SVG 實作，不要使用一串圖片幀。

結構：

進度環使用：

strokeDashoffset =
  circumference * (1 - progress);

視覺要求：

外圈為暗金或米白細線
進度部分為淡金搭配低飽和青綠
不使用霓虹綠
不使用強烈外發光
圓心可以有低透明度太極或氣旋符號
按住時稍微放大至 1.03
放開後恢復正常比例

圓環下方文字：

按住不放

按住後改為：

正在感應靈氣

放開但未完成：

再次按住，繼續感應
五、靈氣粒子系統

不要把靈氣做成固定位置的光球圖片。

使用 Canvas 或 DOM 粒子系統。

建議粒子數：

const MAX_SPIRIT_PARTICLES = 24;

每個粒子：

interface SpiritParticle {
  startX: number;
  startY: number;
  x: number;
  y: number;
  targetX: number;
  targetY: number;
  size: number;
  opacity: number;
  speed: number;
  phaseOffset: number;
}

目標位置是角色丹田附近，而不是整張圖正中央：

const DANTIAN_POINT = {
  x: 0.51, // 相對畫面寬度
  y: 0.56  // 相對畫面高度
};

實際位置需依底圖角色位置微調。

靈氣顯現規則

靈氣效果強度直接由進度控制：

visualIntensity = smoothstep(0, 1, progress);

粒子可見數量：

visibleParticleCount =
  Math.floor(MAX_SPIRIT_PARTICLES * visualIntensity);

透明度：

particle.opacity =
  baseOpacity * visualIntensity;

因此：

進度 0%：幾乎看不到
進度 20%：遠處出現少數微光
進度 50%：竹林與道路旁開始出現靈氣
進度 80%：多數靈氣朝角色靠近
進度 100%：靈氣聚集於丹田附近

不要在開始時就把全部靈氣顯示出來。

六、靈氣靠近角色

每個粒子從畫面外圍或場景深處生成。

例如：

左側竹林
右側石燈附近
遠方山門
地面植物周圍
畫面上方霧氣

移動採用緩動曲線，不要直線等速。

const attraction =
  0.15 + progress * 0.85;

particle.x = lerp(
  particle.startX,
  particle.targetX,
  easeInOutCubic(localProgress)
);

particle.y =
  lerp(particle.startY, particle.targetY, eased) +
  Math.sin(time * particle.speed + particle.phaseOffset) * swayAmount;

靠近丹田後：

不要直接消失
先繞丹田半圈或一圈
再降低透明度
重生於遠處

這樣才能看起來像靈氣被吸引，而不是光點飛進垃圾桶。

七、靈氣流線

除光點外，增加少量細流線。

不要超過：

const MAX_SPIRIT_TRAILS = 6;

流線：

低透明度
淡青白
彎曲
寬度約 1～2 px
由場景邊緣指向角色
隨進度逐漸顯現

可以使用 Canvas Bézier curve 或 SVG path。

不要加入：

巨型旋風
圓形法陣
閃電
爆炸
滿屏發光霧
八、背景灰暗轉溫潤

使用兩層疊合，不直接修改圖片檔。

建議結構：

<div class="scene">
  <img class="background" />
  <div class="gray-filter" />
  <div class="warm-filter" />
  <canvas class="spirit-layer" />
  <div class="ui-layer" />
</div>

初始背景使用較灰暗狀態：

.background {
  filter:
    saturate(0.62)
    brightness(0.78)
    contrast(1.05);
}

隨進度插值到：

filter:
  saturate(0.95)
  brightness(0.94)
  contrast(1.02);

另外疊加淡暖色：

.warm-filter {
  background:
    radial-gradient(
      circle at 51% 55%,
      rgba(214, 190, 126, 0.13),
      rgba(214, 190, 126, 0) 48%
    );
  opacity: var(--progress);
}

不要讓畫面變成鮮豔黃橘色。

變化應是：

由凡人看見的灰暗世界，逐漸顯露原本存在的靈機。

九、竹葉流動

只有一張底圖時，不可能直接讓底圖內每片竹葉活動。

請使用額外的前景粒子模擬：

4～8 片半透明竹葉

竹葉從畫面上方或左右緩慢飄過：

leaf.x += windX * deltaTime;
leaf.y += fallSpeed * deltaTime;
leaf.rotation += rotationSpeed * deltaTime;

可見度隨進度增加：

leafOpacity = progress * 0.45;

竹葉不可太多，避免變成落葉特效。

十、衣帶與角色呼吸

只有完整角色底圖時，不能真正讓衣帶單獨擺動。

Demo 階段使用以下替代方式：

角色呼吸

以角色中心為基準做極小幅度縮放：

.character-layer {
  animation: breathing 3.4s ease-in-out infinite;
}

@keyframes breathing {
  0%, 100% {
    transform: scale(1);
  }
  50% {
    transform: scale(1.004);
  }
}

但這要求角色為獨立透明圖層。

若角色已與背景合成，禁止縮放整張底圖，否則建築和竹林會一起呼吸。

衣帶

正式版應追加透明圖層：

十二、按住、放開與恢復的視覺反應
按住時
進度上升
圓環順時針填滿
粒子數增加
粒子向丹田靠近
背景逐漸恢復色彩
流線逐漸明顯
竹葉輕微飄動
放開時

前 0.25 秒：

畫面維持當前狀態

之後：

進度極慢回退
粒子暫停靠近並略微漂散
背景色彩非常緩慢回灰
圓環緩慢倒退

但不得：

突然熄滅
全部粒子消失
背景瞬間變灰
跳出失敗文字
播放負面音效
再次按住時

立即從目前進度繼續：

粒子重新轉向丹田
流線恢復
進度繼續增加
