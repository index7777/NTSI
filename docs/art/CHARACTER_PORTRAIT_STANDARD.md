# 劇情人物立繪與頭像規格

## 使用原則

- 場景需要交代人物站位、動作或同框比例時，使用完整立繪。
- 對話以文字為主、不需要完整身形時，使用胸像頭像演出。
- 禁止使用程式幾何頭部、圓形剪影或臨時人偶代替已建立的角色。
- 角色名稱直接顯示，不使用【】或其他系統提示括號。
- 遊戲基準畫面為 1200 × 675 橫式；人物站位與頭像顯示必須依橫式 UI 母板驗收。

## 頭像裁切母版

- 母檔：1024 × 1024 PNG，透明背景。
- 安全區：人物頭頂、髮飾及左右髮絲距邊界至少 64 px。
- 構圖：頭頂至胸口或腰帶上緣，臉部中心位於畫面高度 38%～44%。
- 遊戲顯示：桌面基準 176 × 220 px；360 px 手機寬度下不得低於 112 × 140 px。
- 對話人物朝向畫面內側；左右版本不得單純鏡像含有不對稱飾品的角色。
- 頭像不得包含姓名、台詞、框線、背景或文字。

## 表情規格

每套服裝至少建立：

1. neutral：中性待機。
2. serious：嚴肅、教學或警告。
3. gentle：認可、安慰。
4. concerned：擔憂、失敗引導。
5. disappointed：冷淡或操作失誤回饋；只在角色設定適用時製作。

表情版本必須保持臉型、五官位置、髮型、服裝、鏡頭與裁切完全一致，只改眉眼與嘴型。

## 歸檔架構

```text
art/portraits/<character-id>/<outfit-id>/
├─ neutral.png
├─ serious.png
├─ gentle.png
├─ concerned.png
└─ disappointed.png
```

執行版本放在：

```text
game/public/assets/portraits/<character-id>/<outfit-id>/<expression>.png
```

## 服裝版本

- 玩家角色至少區分 novice、qi、foundation；後續新增境界時同步新增。
- NPC 換服裝、身份或境界時建立新的 outfit-id，不覆蓋舊版本。
- 每套新服裝完成後，必須同步補齊該角色劇情已使用的表情。

## 驗收要求

- 先通過角色母板與同框比例驗收，再製作頭像。
- 頭像必須與完整立繪進行臉型、髮型、服裝及配色比對。
- 新人物未完成 neutral 頭像、完整立繪與比例驗收前，不得標記正式完成。
- 表情狀態統一登錄於 game/src/config/characters.ts。
