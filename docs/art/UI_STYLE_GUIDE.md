# 《九劫問仙》橫式遊戲介面母板規格 1.0

正式視覺依據：`art/references/ui/game-interface-style-v1.png`。

切換流程依據：`art/references/ui/scene-card-transition-style-v1.png`。

兩張母板只作為實機畫面配置、UI、UX、HUD 與流程參考，不得直接作為遊戲背景或完整介面圖片使用。

母板中的「宗門事務」與「散步遊歷」僅為排版示意，不屬於實機功能或待開發項目。

## 基準畫面

- 遊戲畫布：1200 × 675，16:9 橫式。
- 桌面與橫向手機共用同一布局，以等比例縮放適配。
- 直向裝置顯示旋轉提示，不另做 9:16 遊戲布局。
- 所有舊 540 × 960 場景與 9:16 圖片標記為 legacy-portrait，只作內容參考。

## 主要布局

- 頂部固定 HUD：玩家頭像、身份、入門日數、卯／午／酉時段、境界層次、修為進度、人物、背包、日誌、設定。
- 左側：縱向修行入口，以活動圖示或小型劇情場景卡呈現。
- 中央：場景、玩家與事件演出的主要空間。
- 右側：NPC、任務、情境操作與對話演出空間。
- 右下角：固定場景移動入口；功能未開放時只顯示系統權限提示。
- 底部：剩餘行動數、臨時狀態或必要系統提示；不得放冗餘教學句。

## 模組比例

- 整體遊戲畫布與主場景：16:9 橫式。
- 場景卡：以約 2:3 的直向卡片放置在橫式畫面中。
- 小遊戲操作模組：以約 3:4 的直向操作區放置在橫式畫面中。
- 結算卡：以約 4:5 的紙卷面板放置在橫式畫面中。
- 直向模組是橫式介面內的獨立元件，不得誤判為整體遊戲改回直式。

## 資產拆分與程式責任

母板內容必須拆分為獨立資產，不得以整張生成介面取代實作：

- 場景背景：卯時、午時、酉時及必要景深圖層。
- 人物：完整立繪、服裝版本、表情頭像與必要動作差分。
- HUD：頂部底板、人物區、時段區、境界修為區及功能入口。
- 面板：紙卷、木框、對話框、名稱牌、日誌、結算與設定底板。
- 圖示：活動、人物、背包、日誌、設定、時段及狀態圖示。
- 按鈕：正常、懸停、按下、停用與選取狀態。
- 進度元件：修為條、時間節點、倒數、分數與操作回饋。
- 文字、數值、列表、存檔、切換、點擊及鍵盤操作全部由程式即時實作。

圖片生成只可用於場景插畫、角色、活動劇情卡或經核准的單一美術原件；禁止生成帶完整 HUD、按鈕、文字或假功能的整張畫面並直接實裝。

## 視覺語言

- 宣紙、舊木、青玉、水墨與克制金色作為統一材質。
- 禁止使用無母板依據的純色矩形、圓形角色、三角形人偶、大型外框線或裝飾橫線。
- 角色名稱直接顯示，不使用方括號或系統提示括號。
- 角色對話、系統提示、活動入口與設定選單使用可辨識的不同層級，但維持同一材質語言。
- 圖片內不得生成繁中文字；所有標題與數值由遊戲 UI 渲染。

## 場景與場景卡

- 場景必須重新以橫式構圖製作，保留左右 UI 安全區與中央人物演出空間。
- 禁止將直式場景拉伸、模糊補邊或單純裁切冒充正式橫式版本。
- 活動劇情場景卡使用約 2:3 的獨立直向卡片構圖，角色眼神、姿勢與表情必須回應活動，不得照抄母板站姿。
- 場景卡下方可由 UI 顯示玩法名稱，不在圖片內生成文字。

## 共用場景進場遮罩

- 純場景切換完成後，先覆蓋全畫面黑色半透明遮罩，再於中央顯示場景名稱與細底線，避免文字被場景明暗吃掉。
- 場景名稱淡入並短暫停留；名稱、底線與遮罩必須一起淡出，全部移除後才開始人物、對話、任務或玩法動畫。
- 由共用 `SceneIntroductionOverlay` 負責，不得在各場景複製另一套動畫。
- 所有場景可選用；日常小遊戲目前預設關閉。雜役處前往砍竹的首次轉場例外啟用，場景名為「小竹峰山下」。

## 共用資訊宣紙

- 上方、下方、左側資訊欄與「小遊戲內 NPC 提示資訊欄」使用乾淨米白宣紙材質。
- 四邊必須是規則直線，不使用撕裂、鋸齒、波浪或不規則毛邊。
- 底板不得包含水墨山水、竹葉、植物、場景插畫或深色水墨底圖；文字、圖示、頭像與分隔內容均由程式獨立疊加。
- 本段只規範小遊戲 HUD。正式劇情演出的台詞底圖仍使用獨立 `xuan-dialogue-panel` 系統，禁止由小遊戲資訊欄資產取代。

## 拼接式核心場景

- 核心大場景禁止將所有可互動內容生成在同一張背景圖中。
- 共用底層拆為遠景宗門／山體、天空、地形、道路、建築與主要陰影；功能區另拆竹林、藥園、問仙石、水井、水面、人物、前景植被、霧與光效。
- 各區塊必須共用同一透視、光源方向、地面材質、墨線粗細與色彩基準；以重疊邊界、遮罩、植被、陰影及霧層消除接圖痕跡。
- 竹林砍伐、藥園乾枯／蟲害、水井結冰／枯水、問仙石發光等狀態必須使用獨立差分，不得重畫或替換整張主場景。
- 拼接完成後需輸出1200×675靜態重組圖及接縫檢查圖；可見硬邊、透視跳動、重影或光源不一致時不得標記完成。

## 卯時、午時、酉時場景版本

- 首圖與核心常駐場景必須各自建立卯時、午時、酉時三個 16:9 橫式版本。
- 核心場景目前包含：首圖、宗門庭院主介面、問仙石、洞天／閉關、築基洞府。
- 同一場景三時段必須維持完全相同的鏡頭、建築、地形、人物錨點與 UI 安全區，只調整天空、光線、霧氣、燈火、陰影及環境氣氛。
- 卯時使用清冷晨光與薄霧；午時提高日照與場景清晰度；酉時使用暖橙夕照、較長陰影與逐步亮起的燈火。
- 檔名統一使用 `<scene-id>-morning-v1`、`<scene-id>-noon-v1`、`<scene-id>-evening-v1`。
- 小遊戲場景目前只製作一個基準時段版本。玩法與美術未正式定案前，禁止批量生成午時與酉時版本。
- 小遊戲三時段版本列為待開發；必須在基準場景、玩法操作與角色演出通過驗收後才能開始。

## 驗收

- 1200 × 675 原尺寸驗收。
- 960 × 540 與 640 × 360 橫向縮放驗收。
- 角色、文字、按鈕不得被安全區、瀏海或手機瀏覽器列遮擋。
- 所有直式場景重製完成前均列為待開發，不得標記橫式 UI 完成。
