# 《九劫問仙》開發交接

最後更新：2026-07-22

## 接手方式

接手後必須遵守根目錄 `AGENTS.md` 的「工作進度溝通」規範：任何生成、下載、建置、測試或工具等待都要說明原因與下一步；禁止在圖片、彈窗或中間產物後無說明停住。

### 不得自行停工（所有接手模型必讀）

目前總目標是完成2026-07-22與其後交辦的全部素材、實際接入遊戲，並依玩家流程做到筑基九層圓滿；雷劫暫緩。完成首頁、單一場景、素材生成、建置、測試、4174驗收、進度回報或階段交付都只是中間檢查點，不代表可以停止。只要總目標尚未完成且存在安全可執行的下一步，回報後必須立即接續下一項。

只有以下情況可以停：使用者明確要求停止／暫停；必須由使用者做出會實質改變成果的選擇；缺少不可替代素材、付費授權或新權限；工具／環境阻塞且替代方案已嘗試仍失敗。停止時必須主動說明原因、正在等待的項目、已嘗試的替代方案與解除後的下一步，不得只留下成果圖、生成器視窗、測試結果或一句進度回報。

本次已發生的禁止案例：完成首圖右上功能列、建置與4174驗收後，將階段成果回報誤當成回合終點而停止。所有後續模型必須知道這不是技術阻塞，而是錯誤停止判斷；正確行為是立即接續問仙石、執事互動、雜役處及後續玩家流程。

問仙石狀態機已開始重構為「逐句文字→執事對話→長按石頭→轉場」。全案不再使用舊直式整圖裁切或相容備案；問仙石頁已建立原生橫式純背景與獨立中性石頭層。正式完成前仍須補齊面向石頭的玩家側背向素材、獨立執事演出層、接觸手部與白光、四種靈根石色／特效層；每層皆須可單獨隱藏、換色與維護。

問仙石與全案正式 VFX 禁止以 CSS、簡單透明度、Phaser 幾何 Graphics 光圈／線條或少量基本粒子交付；現有程式光圈只屬 debug。正式版必須使用授權清楚的商用特效或生成的專用透明素材，分層實作掌心白光、接觸區光、沿裂紋上行靈流、蓄力、完成爆光、轉場霧光，以及四種靈根各自的石色、紋理光與粒子節奏，再由 Phaser 以遮罩、混合模式、粒子系統、時序與鏡頭組合，並在1200×675實機驗收。

本機採雙埠：4173保留給使用者預覽，Codex自動測試固定優先使用4174，必要時使用4175～4178，不得終止使用者的4173程序。

商業遊戲架構與 demo 後搬遷順序已鎖定於 `docs/production/COMMERCIAL_GAME_ARCHITECTURE.md`。目前可以在單一場景內快速完成 demo，但不得把該結構視為正式架構；demo 驗收後依文件分階段拆分，並為所有小遊戲接入 `runSelfCheck()` 與完整流程回歸。

已有 `C:\NTSI` 工作目錄時，從專案根目錄執行 `sync.bat`：

- `[1] Start work`：拉取遠端最新進度，再安裝／更新套件。
- `[2] Ending work`：建置、提交並推送本次工作。
- `[3] Status`：查看分支、未提交變更、Node 與套件狀態。
- `[4] Install dependencies only`：只處理套件，不再拉取。

新電腦第一次接手才使用：

```powershell
git clone https://github.com/index7777/NTSI.git
cd NTSI\game
npm ci
npm run dev
```

正式建置：

```powershell
cd NTSI\game
npm run build
```

完成一項任務後，先更新 `docs/production/HANDOFF.md`，再於專案根目錄執行 `sync.bat` 並選擇 `[2] Ending work`。

## 音樂流程

1. 首圖：`OST 001 NTSI menu`，首次音訊互動後漸入。
2. 選角、取名、問仙石與資質揭示：`OST 0001 NTSI game start`。
3. 點擊「前往雜役處」後：停止Game Start曲，三首日常音樂依序循環並漸入。

瀏覽器會限制未經互動的自動播放，因此Menu曲最早會在玩家首次點擊或按鍵後開始。
