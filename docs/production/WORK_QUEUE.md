## 自循環執行規則

1. 每次開始依序讀取 `AGENTS.md`、`HANDOFF.md`、本檔及當前項目引用規格。
2. 核對 Git、實際程式與 runtime 資產；文件寫完成但專案沒有成果時，狀態退回「未完成」。
3. 只處理編號最前且依賴已滿足的項目，不得挑容易的後段先做。
4. 每階段的劇情、資料、場景、人物、UI／UX／HUD、互動、音訊、VFX、存檔與測試須一起完成。
5. 實作完成後改為「已完成／未驗收」，記錄成果與驗證；此時不得開始下一項。
6. 只有使用者驗收後才能改為「已驗收」並進入下一項。修改意見須改為「退回修改」並逐條加入原項目。
7. 使用者若明確插單，須記錄插單原因與新順序。
8. 上下文壓縮、模型切換或同步後，重新執行第1～3步，不得只靠摘要記憶施工。
9. 發現漏做時退回最早受影響項目，後續項目標記「受前項影響待複查」。
10. 程式修改後必跑 `npm run validate:balance`、`npm run build`；流程測試用4174，必要時4175～4178，不干擾4173。

狀態只使用：`未完成`、`進行中`、`已完成／未驗收`、`退回修改`、`已驗收`、`暫緩`、`阻塞`。
