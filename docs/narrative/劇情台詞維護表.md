## 商業遊戲劇情資料欄位

每一段對話事件必須具備以下欄位，禁止只留下台詞而沒有觸發與結果資料。

| 分類 | 欄位 | 說明 |
|---|---|---|
| 管理 | Event ID | 永久唯一事件編號，發布後不得重複使用 |
| 管理 | Line ID | 每句台詞的永久唯一編號 |
| 管理 | 版本／狀態 | 草稿、待實裝、已實裝未驗收、已驗收、廢止 |
| 流程 | 所屬章節／玩家階段 | 例如 S02 問仙石 |
| 流程 | 觸發方式 | 點擊、進場、自動、數值達標、旗標變更 |
| 流程 | 前置條件 | 境界、任務、性別、日期、數值、必要旗標 |
| 流程 | 重播規則 | 僅首次、每日一次、可重播、不可重播 |
| 流程 | 中斷／略過規則 | 是否可快進、能否跳過整段、中斷後從何處恢復 |
| 角色 | 說話者 ID／顯示名 | 內部角色 ID 與畫面名稱分開記錄 |
| 角色 | 服裝／表情 | outfit-id、expression |
| 角色 | 人像資產 | 1024×1024 胸像或另行核准的全身立繪 |
| 角色 | 人像站位 | 左／右、錨點、安全邊距、朝向 |
| 文字 | 台詞原文 | 經校字的繁體中文正式文本 |
| 文字 | 語氣／情緒 | neutral、serious、gentle 等演出指示 |
| 文字 | 標點與斷行 | 固定標點、強制換行及不可拆詞位置 |

| 演出 | 入場／退場 | 淡入、位移、鏡頭、人物進退場時序 |

## 共用演出規則

- NPC 與玩家姓名直接顯示，不加 `【】`。
- 姓名下方分隔線寬度固定為「姓名字數 + 1」個中文字寬。
- 每句台詞第一個字前固定空兩個全形中文字：`　　`。
- 台詞以逐字方式播放；第一次點擊立即顯示完整當句，當句完整後再點擊才播放下一句。
- 尚有下一句時，在對話卡右下顯示 `（下一句……）`。
- NPC 預設使用同角色、同服裝版本的 1024×1024 透明胸像。
- 胸像位於宣紙對話卡右側；右緣離卡片右緣 2 個中文字寬，下緣與下一句提示保留安全間距。
- 只有劇情另行指定時才使用全身人物版本。
- 修改本檔後，必須同步修改遊戲台詞資料或程式，不得只改文件。
