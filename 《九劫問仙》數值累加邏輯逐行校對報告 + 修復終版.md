# 《九劫問仙》數值累加邏輯修復終版

## 1. 唯一計算原則

本文件是數值的最高優先來源。其他文件如有不同數字，以本文件為準。

- 四維屬性：靈氣、根骨、肉身、神識，分別獨立計算，不得合併成總分後比較。
- 跨大境界時，上一境界的最終屬性定型，成為下一境界的起始基底。
- 跨境成功後不會跌回上一個大境界。
- 小境界突破失敗只降低當前大境界層數，保留全部既有屬性。
- 重修取得的新成長直接累加到既有屬性，不覆蓋、不扣除舊屬性。
- 每項屬性達到目前目標層的上限後，該項本次成長固定為 0。
- 所有乘法結果採無條件進位，最後再依目標層上限裁切。

單項屬性成長公式：

```text
本次有效成長 = ceil(本次隨機成長 × 靈根倍率)

突破後屬性 = min(
  目標層屬性上限,
  目前屬性 + 本次有效成長
)
```

若目前屬性已大於或等於目標層上限：

```text
本次有效成長 = 0
```

這代表玩家在該境界層能取得的能力值已達上限，系統不能再給予能力值；不代表歷史上每一層都曾擲出最大值。

## 2. 靈根倍率

|靈根|倍率|
|---|---:|
|雜靈根|0.6|
|三靈根|0.8|
|雙靈根|1.0|
|單靈根|1.3|

靈根影響單次成長速度，不改變各境界最終硬上限。低靈根可透過多次跌層重修，反覆累加未滿部分，最終抵達相同境界天花板。

## 3. 單輪逐層成長區間

以下均為乘靈根倍率前的單項基礎值。

|境界|1～3層|4～6層|7～9層|單輪最大累加|
|---|---:|---:|---:|---:|
|練氣|2～6|4～11|7～18|105|
|筑基|8～18|14～30|22～45|279|
|金丹|25～45|40～70|60～100|645|

最大累加驗算：

```text
練氣：3×6 + 3×11 + 3×18 = 105
筑基：3×18 + 3×30 + 3×45 = 279
金丹：3×45 + 3×70 + 3×100 = 645
```

上述 105／279／645 是「單輪全部擲出最大值的成長量」，不是境界最終屬性。境界天花板可以高於單輪成長量，玩家須透過跌層重修補足。

## 4. 單輪分段最大成長量

|境界|1～3層|1～6層|1～9層|
|---|---:|---:|---:|
|練氣|18|51|105|
|筑基|54|144|279|
|金丹|135|345|645|

驗算：

```text
練氣1～6：18 + 3×11 = 51
筑基1～6：54 + 3×30 = 144
金丹1～6：135 + 3×70 = 345
```

舊值練氣 84、筑基 234 均作廢。

## 5. 境界九層硬天花板

|境界|靈氣|根骨|肉身|神識|
|---|---:|---:|---:|---:|
|練氣9層|220|180|190|160|
|筑基9層|660|540|570|480|
|金丹9層|1500|1200|1300|1100|

這些是允許多次重修後抵達的最終硬上限，不再宣稱單輪全滿 roll 必然抵達九層天花板。

## 6. 跨境定型與分境界屬性組件

玩家跨境當下的四維屬性分別定型並完整繼承，但不得在新境界補回上一境界的缺口。程式必須分開保存各境界貢獻：

```text
顯示總屬性 = 練氣組件 + 筑基組件 + 金丹組件
```

跨入筑基後，練氣組件永久鎖定；筑基修煉與重修只能增加筑基組件。跨入金丹後，練氣與筑基組件都永久鎖定；金丹修煉只能增加金丹組件。

各境界組件最高貢獻如下：

|屬性組件|靈氣|根骨|肉身|神識|
|---|---:|---:|---:|---:|
|練氣組件上限|220|180|190|160|
|筑基組件上限|440|360|380|320|
|金丹組件上限|840|660|730|620|

驗算：

```text
完美筑基總上限 = 練氣組件上限 + 筑基組件上限
                 = 660／540／570／480

完美金丹總上限 = 完美筑基總上限 + 金丹組件上限
                 = 1500／1200／1300／1100
```

若玩家以未圓滿的練氣屬性跨入筑基，缺少的練氣組件永久保留為本局缺口。即使後來刷滿筑基組件，也不能用筑基成長補足練氣組件。

「筑基一層標準值」不是額外憑空生成的固定數字，而是：

```text
筑基一層標準值 = 已定型練氣組件 + 筑基一層有效成長
```

## 7. 每層上限表的生成方式

程式不得手寫互相矛盾的 1～9 層表。每層上限由配置表生成：

```text
本境界第N層顯示上限 =
  所有已定型前境界組件
  + 本境界第N層組件上限
```

重修時只能比較並增加「目前境界組件」，不能直接拿顯示總屬性補前境界缺口。實際每層組件上限需在平衡測試配置中保存，四維可分別配置；3、6、9層必須通過自動驗算，確保單調遞增，且第9層等於該境界組件上限。

## 8. 跌層重修範例

玩家筑基八層突破失敗，隨機下降 1～3 層，假設掉到筑基六層。所有屬性保持不變。

再次從六層往七層突破成功後：

- 靈氣若已達筑基七層靈氣上限，靈氣增加 0。
- 肉身若未達筑基七層肉身上限，肉身取得一次隨機成長並累加，最多補到七層上限。
- 其餘屬性同樣獨立判定。

反覆跌層與重修會逐步補足未滿屬性，因此低靈根仍有機會抵達境界天花板。
